## Post-launch architecture changes (2026-06) — supersede parts of the plan below

These were done after the original phases and override the relevant earlier steps.
Full rationale in `.claude/rules/decisions.md` and `docs/ARCHITECTURE.md` §7.

- **Intent resolution is now a classifier, not NN over centroids.** Semantic stage =
  per-language logistic-regression head over bge-m3 (`src/classifier/`), with an
  out-of-domain gate using the old centroids. Fixed shared-function-word confusion
  ("vada kodi" vs "bill kodi") and class-imbalance bias (everything → ADD).
  - Retrain: `npx ts-node scripts/train-intent-classifier.ts` (offline, Cloudflare only).
    **Required after any vocabulary change**, then restart the server.
  - Eval: `scripts/run-intent-eval.ts <file>` + `scripts/eval-traps.ts` (held-out).
- **Language scope reduced to 5 active:** hi-IN, kn-IN, mr-IN, ta-IN, te-IN.
  bn-IN, gu-IN, es-ES, ar-AE disabled (`active=false`, embeddings purged from index).
  Toggle with `scripts/toggle-languages.ts on|off <codes>` (off purges embeddings;
  on regenerates them). Demo dropped auto-detect — language is always selected/passed.
- **Vocabulary expanded** for coverage + class balance via
  `scripts/insert-supplement-phrases.ts <file>` (supplements in `scripts/data/`).
  One embedding per term, regenerated on insert.

## Expert Review Provision

Claude-generated vocabulary and eval sets are good enough to ship and demo.
If you later get access to native speakers, corrections can be fed in two ways:

**Option A — Typed corrections**
A native speaker reviews staged entries in Retool and edits any phrase that
sounds unnatural. The corrected phrase goes through the normal IRAL check
and promotion flow. No special tooling needed — it's the existing review UI.

**Option B — Voice recordings (future)**
Record native speakers saying natural phrases. Feed the transcript through
/enrich. If it misses or resolves incorrectly, it surfaces in lexos_misses
and the generation pipeline creates a corrected entry. The reviewer approves it.
No special import needed — the miss loop handles it automatically.

Neither option is required before building or demoing.
